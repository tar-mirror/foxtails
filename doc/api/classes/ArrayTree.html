node_can_have_children!(node)

Set up the node for having children.

Also serves as a hint, for the purposes of icon selection, that this node can have children. This need not be called directly by the user, though you may want to if you have a node without children and want it to be displayed as if it can have children (like an empty folder). It is called internally as a result of a child joining a parent.
